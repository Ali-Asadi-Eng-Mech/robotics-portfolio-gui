# داشبورد مدیریتی پورتفولیوی رباتیک

این پوشه یک وب‌اپ فارسی راست‌چین برای پیگیری پروژه‌های فایل «پورتفولیوی رباتیک» است. متن‌ها و کارت‌ها بر اساس نسخه جدید فایل بازنویسی شده‌اند و نگارش فارسی، ساختار تصمیم‌گیری، ددلاین‌ها، مسئول‌ها و خروجی‌های قابل تحویل تمیزتر شده‌اند.

## اجرا روی سیستم خودتان

برای تست سریع، فایل `index.html` را در مرورگر باز کنید.

برای تست کامل‌تر حالت PWA و Service Worker بهتر است از یک سرور محلی استفاده کنید:

```bash
python -m http.server 8000
```

سپس این آدرس را باز کنید:

```text
http://localhost:8000
```

## انتشار با لینک

برای اینکه مخاطب فقط با یک لینک وارد اپ شود، باید کل پوشه روی یک هاست یا سرویس انتشار قرار بگیرد. ارسال فقط فایل `index.html` کافی نیست، چون اپ به فایل‌های `styles.css`، `app.js`، `data.js`، `manifest.webmanifest`، `sw.js` و `favicon.svg` هم نیاز دارد.

روش‌های ساده انتشار:

1. کل پوشه را روی GitHub Pages منتشر کنید.
2. کل پوشه را روی Netlify یا Vercel آپلود کنید.
3. لینک خروجی را برای مخاطب بفرستید.

پس از انتشار، مخاطب لازم نیست پوشه را داشته باشد؛ فقط لینک را باز می‌کند و نسخه اصلی اپ از روی هاست بالا می‌آید.

## فونت فارسی

در CSS، فونت اصلی روی `B Nazanin` تنظیم شده است. اگر این فونت روی سیستم مخاطب نصب نباشد، مرورگر از فونت‌های جایگزین استفاده می‌کند. برای نمایش تضمینی B Nazanin روی همه سیستم‌ها باید نسخه وب‌فونت دارای مجوز قانونی روی هاست قرار داده شود.

## قابلیت‌ها

- داشبورد مدیریتی با KPI
- کارت‌های پروژه با مسئول، ددلاین، اولویت، خروجی و ریسک
- برد کانبان با Drag & Drop
- چک‌لیست هر تسک
- یادداشت پیگیری برای هر تسک
- خروجی CSV و JSON
- چاپ / ذخیره PDF از مرورگر
- طراحی RTL فارسی
- ذخیره وضعیت‌ها در LocalStorage مرورگر
- PWA آماده نصب روی موبایل/دسکتاپ

## محدودیت مهم

این نسخه دیتابیس ندارد. یعنی تغییرات هر کاربر فقط در مرورگر همان کاربر ذخیره می‌شود. برای استفاده تیمی واقعی، باید نسخه بعدی به دیتابیس مثل Supabase یا Firebase وصل شود.

## تغییرات این نسخه

- بازنویسی متن‌ها بر اساس فایل جدید «پورتفولیوی رباتیک».
- اصلاح ساختار نگارشی کارت‌ها، ریسک‌ها، تصمیم‌ها و برنامه اقدام.
- تنظیم فونت اصلی فارسی روی B Nazanin.
- اضافه شدن اولویت‌های جدید مثل «در انتظار پاسخ»، «نیازمند تصمیم دکتر» و «نیازمند پیگیری قراردادی».
- حفظ رفع باگ Modal و دکمه بستن.
- تغییر کلید ذخیره محلی برای جلوگیری از تداخل با نسخه قبلی.


## قابلیت جدید

- ثبت تاریخ و زمان آخرین به‌روزرسانی سایت از داخل تب «ویرایش و ذخیره»

- ذخیره اطلاعات نسخه در پشتیبان کامل JSON
- اضافه شدن راهنمای معماری ربات تلگرام در `README_TELEGRAM_BOT_FA.md`
