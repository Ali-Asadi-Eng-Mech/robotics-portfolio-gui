# مسیر درست برای ربات تلگرام

این داشبورد روی GitHub Pages یک سایت استاتیک است؛ یعنی سرور، دیتابیس و محیط امن برای نگهداری توکن ربات ندارد. بنابراین نباید توکن BotFather را داخل `app.js` یا `data.js` قرار دهید، چون هر کسی می‌تواند آن را از سورس سایت بردارد.

## معماری پیشنهادی

1. ساخت Bot با BotFather و دریافت `BOT_TOKEN`.
2. ساخت یک بک‌اند کوچک، ترجیحاً Cloudflare Worker، Vercel Function، Netlify Function یا Google Apps Script.
3. ذخیره توکن در Environment Variable همان بک‌اند، نه داخل فایل‌های سایت.
4. گرفتن `chat_id` افراد بعد از اینکه هر فرد یک بار به ربات پیام `/start` بدهد.
5. اضافه کردن فیلد `telegramChatId` یا `telegramUsername` برای هر مسئول در داشبورد.
6. وقتی تسک جدید یا پیام مدیریتی ثبت شد، فرانت‌اند درخواست را به بک‌اند بفرستد و بک‌اند با Telegram Bot API متد `sendMessage` را صدا بزند.

## نکته مهم

ارسال مستقیم پیام تلگرام از GitHub Pages بدون بک‌اند امن نیست. نسخه فعلی فقط برای داشبورد رمزدار و ویرایش محلی مناسب است. برای اعلان تلگرام واقعی باید مرحله بعد، یک بک‌اند کوچک بسازیم.
